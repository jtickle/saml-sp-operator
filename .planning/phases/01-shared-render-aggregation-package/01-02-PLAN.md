<objective>
Build the two pure byte-level primitives every renderer needs: `collapseEmptyElements` + XML prolog prepend (`xmlformat.go`), and the length-prefixed `sha256` config hash (`confighash.go`, RENDER-09/D-08). These have no dependency on the type model, so this plan runs in Wave 1 in parallel with plan 01.

Purpose: `encoding/xml` never self-closes empty elements and never emits the `<?xml?>` declaration — reproducing the spike fixtures byte-for-byte is impossible without the collapse+prolog pass. The config hash is what gates pod rollout in Phase 2, and it must cover all three rendered files.
Output: `xmlformat.go` (collapse + prolog) + an empirical-stdlib-behavior test, and `confighash.go` (Hash) + determinism/change-sensitivity tests.
</objective>

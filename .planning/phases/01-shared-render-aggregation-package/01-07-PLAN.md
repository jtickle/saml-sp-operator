## Test-mechanism decision (RESEARCH Open Question 3)

Adopt `github.com/testcontainers/testcontainers-go` (RESEARCH's recommendation) for the load test rather than a hand-rolled `os/exec` docker wrapper: it provides guaranteed container cleanup (even on test panic) and `wait.ForLog`-style startup detection, which matter for a CI-gated test. It is a Docker-only library with no Kubernetes surface, so importing it behind the `shibdload` build tag does NOT violate D-01/D-02 (the hermetic package still has zero k8s and zero container deps — verify with `go list -deps` on the non-test package). Per RESEARCH's Package Legitimacy Audit both `testcontainers-go` (official org, 1,700+ importers) and `go-cmp` (Google) are Approved — no [ASSUMED]/[SUS]/[SLOP] packages, so no blocking legitimacy checkpoint is required; re-confirm the exact current version via `go list -m -versions` at execute time.

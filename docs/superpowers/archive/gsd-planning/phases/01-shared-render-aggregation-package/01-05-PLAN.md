## Scope note (RESEARCH Pitfall 6)

The repo-root `nginx.conf` is a pure FastCGI→HTTP adapter for the Traefik-ForwardAuth model — it has NO `more_clear_input_headers`/clear-list directive and no `headers_more`/`ngx_http_shibboleth` module. RENDER-08's clear-list is therefore a pure-Go VALUE computation in `clearlist.go` only; do NOT add a clear-list directive to the nginx.conf template and do NOT golden-file-compare `ClearList` against nginx.conf (there is nothing in that file to compare against).

| CRD strings → nginx.conf directives | text/template performs NO auto-escaping; a hostile CRD string could break nginx directive structure |
| edge → app (identity headers) | The clear-list is what strips client-spoofable Variable-* headers at the edge before the authorizer sets the trusted ones |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-05-01 | Tampering | Hostile CRD string breaks nginx.conf directive structure (no template auto-escaping) | high | mitigate | validateHostname allowlist (^[a-zA-Z0-9.-]+$) applied to every CRD-derived string BEFORE tmpl.Execute; reject at render time rather than escaping after the fact (RESEARCH Code Examples). |
| T-05-02 | Spoofing | Client spoofs a Variable-* identity header that the app trusts | medium | mitigate | RENDER-08 clear-list enumerates (Traefik) / globs (nginx) the exported identity headers so they are cleared at the edge before the authorizer sets the trusted values; the render package computes the correct clear-list value per attachment model. (Actual edge enforcement is Phase 5 Middleware emission / the future standalone tool.) |

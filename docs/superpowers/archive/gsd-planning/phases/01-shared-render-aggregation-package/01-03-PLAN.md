<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CRD string fields → shibboleth2.xml | Hostile/odd AppBinding/SPConfig strings (entityID, hostname) marshal into XML element/attribute values |
| render self-URL → shibd request authority | The RequestMap <Host> key must match the reconstructed request authority (scheme+port) or the auth gate fails open |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege (auth bypass) | RequestMap <Host> missing explicit scheme+port | high | mitigate | RENDER-05: always emit scheme AND port even on default ports (spike fix N). A bare-hostname Host auto-expands to standard ports only, so a non-standard-port request matches nothing, requireSession is never applied, and the authorizer returns 200 for unauthenticated traffic (fail-OPEN). TestHostSchemePort asserts the attributes are always present. |
| T-03-02 | Tampering | XML injection via hostile CRD string fields (entityID, hostname, REMOTE_USER) | high | mitigate | encoding/xml struct marshaling auto-escapes every string routed through a normal struct-tag field (D-05). Adversarial coverage is exercised in plan 06 (TestInjectionSafety). |
| T-03-03 | Denial of Service | `--` in a CRD string routed into an XML comment aborts the whole render | medium | mitigate | Use the struct-tag `,comment` path (rejects any `--`) AND strip at the input layer (strings.ReplaceAll(v,"--","-")) so one hostile field cannot abort config generation for the entire SP (D-05). |
</threat_model>
